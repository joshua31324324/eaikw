# Resource Library – Job Club

## Purpose

The Resource Library provides NJIT students with curated, student‑friendly AI career tools. It ensures that guides are accessible, well‑formatted, optimized for SEO, and reviewed by humans even if AI‑assisted in drafting.

---

## Resource Library Page

- **Section Title:** Career Resources
- **Resource Cards (listing view):**
  - Title
  - Short description
  - CTA Button: [View Resource]
- **Minimum Content:**
  - At least 2 written guides stored as Sanity `resource` documents.
- **Recommended Examples:**
  - How to Optimize Your LinkedIn for AI Jobs
  - AI Consulting Portfolio Starter Guide
  - How to Build a GitHub Profile That Gets You Interviews
  - How to Set Up Calendly + Zapier for Client Meetings
- **Actions:**
  - [View All Resources] button
  - Filter by category (career branding, technical setup, automation tools)

---

## Resource Detail Page

- **Content:**
  - Title
  - Author attribution
  - Full guide content (formatted for readability)
  - Links to templates or external references
- **Formatting Requirements:**
  - Headings and subheadings for scannability
  - Lists and tables for clarity
  - Accessible markup (ARIA labels, alt text for images)
- **SEO Requirements:**
  - Metadata (title, description, keywords)
  - Semantic HTML structure
  - Internal linking to related resources

---

## Sanity CMS Content Model

- **resource**
  - Title
  - Description
  - Content body (Markdown/HTML)
  - Author reference
  - Tags (career branding, portfolio, automation, etc.)
- **author**
  - Name
  - Bio
  - Links (LinkedIn, GitHub, portfolio)

---

## Compliance

- **Accessibility:** WCAG‑compliant formatting, ARIA support, alt text for images.
- **SEO:** Optimized metadata, semantic headings, internal linking.
- **Human Review:** AI‑assisted drafting but reviewed by staff or mentors before publishing.
- **GDPR:** Cookie banner and analytics consent required before tracking resource views.

---

## Footer

- **Links:** Privacy policy, accessibility, contact.
- **GDPR:** Cookie banner with Accept, Reject, Preferences; analytics load only after consent.
- **Accessibility:** ARIA labels, semantic landmarks, keyboard navigation confirmed.
